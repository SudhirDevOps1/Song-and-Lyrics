Build a highly advanced, modern, and interactive Music & Lyrics Sync Web Application (similar to Spotify/Apple Music lyrics feature but with video support). 

# Technology Stack
- **Framework:** React 18+ initialized with Vite (`npm create vite@latest`).
- **Styling:** CSS Modules or Styled Components (with CSS Variables for theming). Implement a dark-mode Glassmorphism theme.
- **State Management:** React Context API or Zustand (recommended for handling complex player state without excessive re-renders).
- **Libraries:** `react-youtube` (for YouTube API), `framer-motion` (for smooth UI animations - optional but recommended).

# 1. UI/UX & Architecture
- **Design System:** Deep space gradients (`#0f0c29` to `#302b63`). 
- **Glassmorphism:** Use `backdrop-filter: blur(20px)` and semi-transparent backgrounds (e.g., `rgba(20, 20, 30, 0.6)`) with subtle borders.
- **Component Structure:**
  - `<App />`: Main layout (CSS Grid).
  - `<Sidebar />`: Contains `<Playlist />` and `<LyricEditor />`.
  - `<MainScreen />`: Contains `<LyricsScroll />`, `<UnifiedPlayer />`, and `<AudioVisualizer />`.
  - `<SettingsPills />`: Customizable toggle buttons.

# 2. Application State (Zustand or Context API)
Maintain a global state containing:
- `songs`: Array of song objects.
- `currentSongIndex`: Integer pointing to active song.
- `isPlaying`: Boolean.
- `currentTime`: Float (managed via `requestAnimationFrame` or fast interval, consider keeping this in a mutable `useRef` to prevent massive re-renders, and only trigger re-renders when the `activeLyricIndex` changes).
- `activeLyricIndex`: Integer.
- `preferences`: Object { speed, animation, alignment, waveType, layout }.

# 3. Audio & Video Player Engine (UnifiedPlayer)
- **YouTube Integration:** Use `react-youtube`. Bind the player instance to a `useRef`.
- **Local Audio Integration:** Use a hidden `<audio>` element bound to a `useRef`.
- **Unified Controls:** Create custom Play, Pause, Next, Prev, and a seek bar `<input type="range">`. 
- **Time Tracking:** Create a custom hook `usePlaybackTime` that queries `playerRef.current.getCurrentTime()` every 50ms using `requestAnimationFrame`.

# 4. Advanced Lyrics Parsing & Sync Logic (<LyricsScroll />)
- **Regex Parsing:** Build a parser for timestamps. Format: `[mm:ss.s] lyric text` or `(0:12 - 0:16): lyric text`.
- **Synchronization Algorithm:**
  - In a `useEffect` hooked to the playback timer, calculate the `newActiveIndex` by iterating backwards through the parsed lyrics array to find the highest index where `currentTime >= lyric.time`.
  - **REACT OPTIMIZATION:** Only update the state `setActiveLyricIndex(newIndex)` if `newIndex !== activeLyricIndex` to prevent infinite re-renders.
  - **Auto-Scrolling:** Use a `useRef` array for lyric line DOM nodes. When `activeLyricIndex` changes, call `lineRefs.current[activeLyricIndex]?.scrollIntoView({ behavior: 'smooth', block: 'center' })`.
- **Karaoke Mode:** If enabled, calculate the duration between `lyric[i]` and `lyric[i+1]`. Pass the progress percentage to a `<KaraokeWord />` sub-component to highlight words sequentially.

# 5. Song Editor & Data Persistence (<LyricEditor />)
- Provide inputs for Title, Artist, and a `<textarea>` for raw timestamped lyrics.
- **Apply Action:** When the user clicks "Apply", parse the text into an array of objects `{ time, text, color }`, update the global state, and save to `localStorage`.
- **Data Persistence:** Use a `useEffect` to watch the `songs` state and `localStorage.setItem('songs', JSON.stringify(songs))` on every change.
- **Server Sync:** On mount (`useEffect` `[]`), fetch `songs.json` from the `public` folder and merge it with local storage data using unique `id`s.

# 6. Canvas Audio Visualizer (<AudioVisualizer />)
- Create a `useAudioVisualizer(audioRef, canvasRef, type)` hook.
- Initialize an `AudioContext` and `AnalyserNode` when the user clicks "Play" (to bypass browser autoplay restrictions).
- Draw responsive waveforms (Bars, Curves, or floating Dots) on the `<canvas>` based on `analyser.getByteFrequencyData()`. Note: This only works for local HTML5 `<audio>`, not YouTube iframes due to CORS.

# 7. Critical React Edge Cases & Bug Prevention
- **Stale Closures:** Avoid stale state inside intervals by using `useRef` for latest state values, or rely on `requestAnimationFrame` passing the current time.
- **Duplicate Event Listeners:** React's synthetic events handle this, but avoid attaching multiple watchers to the Audio element. Use standard `<audio onPlay={...} onPause={...}>` props.
- **Fullscreen Mode:** Use the standard browser Fullscreen API on a wrapper `<div>`. Ensure React state accurately reflects fullscreen status by listening to `'fullscreenchange'` events.
- **Dependency Arrays:** Be very strict with `useEffect` dependencies to prevent memory leaks or continuous fetching loops.
